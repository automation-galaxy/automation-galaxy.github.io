Workload Placement vROPS ( STORAGE FOCUSED )

  1. Setup vROPS
  2. Configure vRA in vROPS
  3. Configure vCenter in vRA
  4. Configure vCenter in vROPS
  5. Edit the cloud zone/s that you want to use workload placement
    1. Modify the cloudzone in
      1. Infrastructure -> cloud zones
        1. Open the desired zone and modify the placement policy. To use vROPS placement, this needs to be set to ADVANCED.
  6. For datastore and datastore grouping, you need to configure a storage profile in vCenter and tag all datastore/ds clusters in that policy that you want vROPS to consider for placement. vRA 8 will only pick one storage profile as a valid storage target for vROPS to choose from, so only one storage profile is supplied to vROPS regardless of tagging.
  7. Once you have created a storage policy in vCenter and a data collection has completed on the cloud account, you can move on and create a Storage profile.
    • The vRA storage profile needs to be configured with the vCenter storage policy that you created.
    • You also need to make sure you DO NOT SELECT ANY VALUE FOR 'Datastore / cluster'.
    • You can also configure required Capability tags.
  8. In vROPS you need to browse to the in-use policy and modify it with the capacity constraints you require:
    1. In vROPS
      1. Configuration -> Policies: Select your active policy.
    2. Select EDIT POLICY in the top right corner.
    3. Select capacity tile.
    4. In the Select Object Type, input Datastore to select the vSphere connector Datastore object.
    5. Click the lock beside the Capacity Remaining and set the critical marker to the % you require (this use case is 75%).
    6. Scroll down and save.

Now you can deploy your cloud template and vROPS will select a valid datastore that is not above the datastore threshold you specified in the vROPS policy.
